# pull-leyu-journal

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库旨在收集、整理并托管若干独立的 HTML 页面。这些页面彼此独立，不隶属于任何特定网站或域名，主要用于信息存档、实验性展示或静态内容发布。

## 目录说明

```
.
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 页面所需的公共资源（如 CSS、JS、图片等）
├── index.html      # 仓库首页 / 页面索引
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件对应一个独立的 HTML 页面，命名遵循 `page-xxx` 或 `xxx.html` 格式。
- **assets/**：存放多个页面可能共用的样式、脚本或媒体文件，避免重复引用。
- **index.html**：可选，用于提供页面导航列表或简要说明。

## 页面归档说明

- 每个 HTML 页面均为完整、自包含的文档，可独立打开浏览。
- 页面内容可能包含文本、图片、交互组件等，具体功能取决于各页面自身的实现。
- 归档页面不保证全部保持最新或持续更新，部分页面可能仅用于历史记录。

## 维护说明

- 欢迎通过 Pull Request 提交新的 HTML 页面或改进现有页面。
- 提交新页面时，请将其放置在 `pages/` 目录下，并确保文件名具有辨识度。
- 公共资源请优先放置于 `assets/` 目录，并在页面中通过相对路径引用。
- 请勿在页面中包含任何形式的广告、跟踪代码或恶意脚本。
